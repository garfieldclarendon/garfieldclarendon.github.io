Detailed Description

API handler class for a LCS device.

{get} /api/device_list:serialNumber,moduleID,controllerID,classCode,deviceID Get a list of devices GetDeviceList Device

{Number} [serialNumber] filter device list by a specific controller's serial number. {Number}[moduleID] filter device list by a specific controller module's ID. {Number} [controllerID] filter device list by a specific controller's ID. {Number} [classCode] filter device list by a specific device classification. {Number} [deviceID] filter device list by a specific device ID. Returns a list of devices. If no parameters are supplied, all devices are returned. {Number} deviceClass Device classification. {String} deviceDescription Device's description. {Number} deviceID Device's ID. {String} deviceName Device's name {Number} deviceState Device's current state. Example usage: http://localhost:8080/api/device_list?controllerID=30 {json} Success-Response: HTTP/1.1 200 OK [{ "deviceClass": "5", "deviceDescription": "", "deviceID": "19", "deviceName": "GK Mine Signal", "deviceState": 0, } ] {get} /api/device_property_list:deviceID Get deviceProperty entries for a given device GetDevicePropertyList Device

{Number} deviceID device's ID. Returns a list of deviceProperty entries for a given device.. {Number} deviceID Device's ID. {Number} id deviceProperty's id. {Number} key Device Property key. {Number} value Device Property value. Example usage: http://localhost:8080/api/device_property_list?deviceID=1 {json} Success-Response: HTTP/1.1 200 OK [{ "deviceID": "1", "id": "243", "key": "MOTORPIN", "value": "1" }, { "deviceID": "1", "id": "244", "key": "INPUTPIN", "value": "1" } ] {get} /api/module_device_port_list:deviceID,moduleID Get module/port entries for a given device GetModuleDevicePortList Device

{Number} [deviceID} device's ID. {Number} [modleID] controllerModule's ID. Returns a list of moduleDevicePort entries for a given device or module. This table cross-references a device to one or more modules. This allows a signled device entry to be used with multiple modules; useful for PanelInputDevice and PanelOutputDevice, etc. {Number} id moduleDevicePort's ID. {Number} deviceID device's ID. {Number} controllerModuleID controllerModule's id. {Number} port port the device is connected to. Example usage: http://localhost:8080/api/device_property_list?deviceID=1 {json} Success-Response: HTTP/1.1 200 OK [{ "deviceID": "1", "id": "58", "controllerModuleID": "4", "port": "0" } ] {get} /api/send_device_config:deviceID Reset device's configuration DeviceConfigReset Device

{Number} [deviceID] The device's id. If 0 or excluded, all devices will reset their configuration data. Sends a SYS_RESET_DEVICE_CONFIG broadcast UDP message instructing the device(s) to re-download its configuration data. Example usage: http://localhost:8080/api/send_device_config?deviceID=1 {get} /api/create_device:deviceClass Create a device entry CreateDevice Device

{Number} deviceClass The classification of the device's to be created. Creates a new, blank device entry. In addition to the device table entry, any required deviceProperty entries are also created. {Number} controllerModuleID Controller Module's ID to which the device is connected. {Number} deviceClass Device's classification populated with the supplied deviceClass. {String} deviceDescription Device's description. {String} deviceName Device Device's name. {Number} id Device's new ID. {Number} port/pin Port to which the device is connected. This example creates a new Turnout device entry. In addition to the device table entry, two entries are also added to the deviceProperty table; MOTORPIN and INPUTPIN with values set to 0: http://localhost:8080/api/create_device?deviceClass=1 {json} Success-Response: HTTP/1.1 200 OK [{ "controllerModuleID": "0", "deviceClass": "1", "deviceDescription": "", "deviceName": "", "id": "83", "port": "0" } ] {get} /api/notification/device Device Status Change DeviceStatusChangeNotification APINotifications off

Notification message sent when a device's state changes. {String} url Notification url. {Number} deviceID Device's ID. {Number} deviceState Device's new state {Number=0,1} locked Device's locked state. 0 = unlocked, 1 = locked. {json} Success-Response: { "url": "/api/notification/device", "deviceID": "1", "deviceState": "2", "locked": "0" }APIDevice This class handles basic API requests for a device. APISignal, APITurnout and APIRoute handle device-specific requests.
